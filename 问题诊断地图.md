# 出海投放 AI 军师：问题诊断地图

这份地图用于扩展 Demo 的提问入口。当前知识库不是只能问 ROI，它覆盖的是“投放策略 + 素材文案 + 技术落地 + 风控踩坑 + 复盘案例”的组合问题。

## 1. 投放策略类

- ROI 小于 1 持续两天，应该关停还是降预算？
- 广告计划突然掉量，是素材疲劳、受众枯竭还是预算问题？
- CTR 高但 CVR 很低，应该优先查素材、落地页还是人群？
- 什么时候该放量，什么时候该止损？
- TOFU、MOFU、BOFU 分别应该怎么做人群排除？
- 再营销应该投给访问过网页但未转化的人，还是连接过钱包/Lead 的人？
- 预算加太快导致学习期波动，应该怎么恢复？

## 2. 素材与文案类

- FB 爆款五步法文案怎么写？
- Hook 不够强，怎么改成更高 CTR 的开头？
- 素材疲劳应该看哪些信号？
- 素材测试表应该怎么判断胜出素材？
- 一个产品要准备哪些素材角度？
- 文案里痛点、信任背书、CTA 应该怎么组织？
- 落地页 Hero Section 怎么写得更高转化？

## 3. 技术落地类

- Pixel、CAPI、事件回传应该怎么配置才干净？
- 动态参数路由里的 pid、goal、segment 分别承担什么作用？
- 前端怎么过滤低质量点击，避免污染像素？
- 后端 CAPI 怎么做降噪和防刷？
- LCP、CLS、TBT 超预算时先修哪里？
- 静态页面怎么升级成动态落地页引擎？
- A/B 测试权重看板应该怎么设计？

## 4. 风控与踩坑类

- 半夜空烧怎么设置自动拦截规则？
- ROI 破位时用什么熔断规则？
- 自动化规则怎么避免误杀好计划？
- 账号、BM、Pixel 怎么分层降低风险？
- 投放资产异常时先看哪些风控信号？
- 备用像素、保险箱像素、主像素分别怎么用？
- 哪些动作容易污染账户权重或像素数据？

## 5. 复盘分析类

- 每日投放复盘应该看哪些字段？
- 今天亏损应该归因到素材、受众、落地页还是技术链路？
- 一条计划从拉新到收割，复盘时怎么拆？
- 怎么把一次失败投放变成下次测试假设？
- 周复盘要沉淀哪些 SOP？

## 6. 组合诊断类

- ROI 下滑但 CTR 没变，是落地页问题还是支付/事件问题？
- 点击成本下降但成交没提升，可能哪里出了问题？
- 新素材首日数据不错，第二天掉得很快，要不要继续烧？
- 预算上调后数据变差，是学习期波动还是素材逻辑失效？
- CAPI 配好后 ROI 反而变差，应该怎么排查？
- 账户频繁异常时，是代理环境、资产结构还是规则误触？

## 7. 真实口语问法

这些是客户真正会说的话，需要映射到上面的诊断词：

- 钱一直烧但是不出单咋办？
- 今天广告感觉废了，还要不要继续跑？
- 点击挺多，就是没人买，哪里有问题？
- 素材是不是疲劳了，怎么看？
- 怎么突然没量了？
- 预算加了以后数据反而变差，是不是加太猛？
- 号是不是有问题，感觉被限流了？
- Pixel 数据乱了，回传是不是不准？
- 半夜又烧钱了，能不能自动拦住？
- 我不知道问题在哪，你帮我排查一下。

## 首批试点建议

演示时不要让客户自由乱问，先准备 20-30 个高频问题按钮。客户看到“它能识别我每天真的遇到的问题”，才会愿意继续试用。
